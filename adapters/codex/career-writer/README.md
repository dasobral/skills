# Career Writer

Evidence-grounded CV, cover-letter, and application-document workflows.

## Daily workflow

1. Use `career-documents` to identify the audience, role, format, and selection criteria.
2. Extract only supported experience and credentials from user-provided sources.
3. Draft, edit, and check traceability, tone, length, and formatting.
4. Ask the user to verify every factual claim before submission.

## Triggers

Use for CV or résumé revisions, cover letters, professional biographies, application responses, and audience-specific career materials.

## Required inputs

- Target role, organization, audience, prompt, and deadline.
- Verified employment, education, skills, accomplishments, and contact details.
- Required format, length, language, and claims that must not be made.

## Artifacts

- Draft and final career documents.
- Evidence/claim checklist and unresolved fact questions.
- Formatting or tailoring notes for the target audience.

## Agent authority

The document editor may rewrite and recommend emphasis. It cannot invent credentials, verify facts independently, submit applications, or install its template automatically.

## Deterministic checks and agent decisions

Length, required sections, formatting, spelling, and source-field presence are deterministic checks. Tone, emphasis, ordering, and persuasive framing remain editorial judgments.

## Data guarantees

Drafts must be based on supplied facts and clearly flag missing evidence. No hook runs and the plugin does not submit or transmit documents.

## Limitations and non-claims

This plugin does not guarantee factual accuracy, hiring outcomes, legal compliance, recruiter preferences, or compatibility with every applicant-tracking system.
