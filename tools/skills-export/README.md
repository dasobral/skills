# skills-export

Export the portable skill **core** to Cursor plugins, Claude Code bundles, or Codex bundles.

## Quick start (no install)

From repo root:

```bash
chmod +x bin/skills-export
./bin/skills-export validate
./bin/skills-export sync cursor      # regenerate plugins/cursor/
./bin/skills-export export all       # write dist/cursor, dist/claude, dist/codex
```

## Install (optional)

```bash
pip install -e tools/skills-export
skills-export validate
```

## Commands

| Command | Purpose |
|---------|---------|
| `validate` | Check manifest ↔ core skills (Agent Skills conventions) |
| `list` | List plugins and skills |
| `sync cursor` | Regenerate `plugins/cursor/` from `core/` + `adapters/cursor/` |
| `export cursor` | Write Cursor plugins to `dist/cursor/` |
| `export claude` | Write `.claude/skills/` bundles + flat export |
| `export codex` | Write `.agents/skills/` bundles + flat export |
| `export all` | All three platforms |
| `ingest` | Landing → core |
| `maintain` | Ingest + validate + sync Cursor + export Claude/Codex |

## Architecture

```
core/
  manifest.yaml          # plugin ↔ skill mapping (source of truth)
  skills/<name>/         # portable SKILL.md + references (Agent Skills standard)
  references/            # platform-paths, platform-orchestration

adapters/
  cursor/<plugin>/       # agents, hooks, rules, plugin.json overlay
  claude/                # export templates (bundles generated by tool)
  codex/

plugins/cursor/          # generated Cursor plugins (sync cursor)
dist/                    # generated Claude/Codex (+ optional Cursor) exports
scripts/cron/            # cron wrapper for maintain
```

**Edit skills in `core/skills/` only.** Run `sync cursor` before committing plugin changes.

Cursor is the committed, ready-to-use target. Claude/Codex are produced under `dist/` when you run maintain or `export`.
